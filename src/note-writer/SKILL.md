# note-writer — ライティングインテリジェンス

note.com/Zenn向け記事執筆の総合ガイド。14記事タイプ、25タイトル公式、25表現技法、5作家別文体を収録。8ドメインのCSVデータベースをBM25検索エンジンで横断検索可能。

## When to Apply

このスキルを参照するタイミング：
- 記事の企画・構想を練るとき
- タイトルを作成・最適化するとき
- 書き出しや構成を設計するとき
- 特定の作家スタイルで書くとき（岸田奈美・椹野道流・さすらい等）
- 公開前の最終チェック（スキ数予測・ハッシュタグ選定）

### Critical Rules（必ず守る）

1. **タイトルで好奇心を刺激する** — 「え？なぜ？」と思わせる
2. **最初の3行で読者を引き込む** — 好奇心・反直感・親密感のいずれか
3. **具体的なシーン/数字を使う** — 抽象表現を排除
4. **結びに力を込める** — 尻すぼみ厳禁
5. **表の両端竖线を省く** — note.comのレイアウト規則

## Workflow

### Step 1: 記事タイプを決める

```bash
python .claude/skills/note-writer/scripts/search.py "[素材のキーワード]" --domain article-types
```

素材（個人体験、業界知識、技術プロジェクト等）に応じて最適な記事タイプを選択。

### Step 2: タイトルを作成する

```bash
python .claude/skills/note-writer/scripts/search.py "[記事タイプ] タイトル" --domain titles
```

タイトル公式に当てはめて、好奇心を刺激するタイトルを作成。

### Step 3: 構成を設計する

```bash
python .claude/skills/note-writer/scripts/search.py "[記事タイプ] 構成" --domain structures
```

記事タイプに対応する構成テンプレートを取得し、各セクションの内容を設計。

### Step 4: 表現技法を適用する

```bash
python .claude/skills/note-writer/scripts/search.py "[技法名 or 記事タイプ]" --domain techniques
```

自虐ユーモア、対比構造、会話テクニック等の技法を記事に適用。

### Step 5: 公開準備

```bash
python .claude/skills/note-writer/scripts/search.py "スキ数 予測" --domain publishing
python .claude/skills/note-writer/scripts/search.py "ハッシュタグ [カテゴリ]" --domain publishing
```

スキ数予測スコアリングでセルフチェック。最適なハッシュタグと投稿時間を選定。

## Writing Templates

### IT告白系テンプレート

```markdown
# [対象者]の皆様、[問題]を[動詞]のは、私です

正直に言います。[告白の内容]。

嘘です。[本当のこと]。

## なぜそうなったのか

[具体的なスケジュール/日常描写]

## 「[キャッチフレーズ]」という地獄

[キャッチーな名前をつけた苦しみの描写]

## これは私だけの問題ではない

[構造的な問題の指摘]

## [対象者]の皆様へのお願い

[建設的な提案（Win-Win）]

---
[最後の懺悔と希望]
```

### エッセイ系テンプレート（岸田奈美式）

```markdown
# [深刻な状況], [意外な比喩]

[短い一文で日常の異変]。

[背景説明（最小限）]

[問題発生 — 深刻そうな出来事]

[会話の噛み合わなさ（ズレの連続）]

---

[転換点 — 予想外の展開]

[感動のクライマックス]

---

[オチ — 余韻を残す短い一文]
```

### 日記系テンプレート（椹野道流式）

```
タイトル: たぶん誰も読まない日記（[キーワード]）
本文（200-350字、改行なし1段落）:
[書き出し1文]。[経緯・背景]。[エピソード]。[猫/母/仕事/体調への言及]。[脱線]。[軽い感想 or 結び]。
ハッシュタグ: #エッセイ #日記 #[テーマ関連]
```

## Pre-Publication Checklist

### 必須チェック（全記事タイプ共通）

- [ ] タイトルは好奇心を刺激するか？
- [ ] 最初の3行で読者を引き込めるか？
- [ ] 具体的なシーン/数字/事例があるか？
- [ ] 結びは力強いか？（尻すぼみでない）
- [ ] 誤字・文法ミスがないか？
- [ ] ハッシュタグを3-5個追加したか？
- [ ] 公開時間は最適か？（平日12-13時/20-22時、週末10-12時/15-17時）

### 記事タイプ別チェック

**IT告白系追加チェック:**
- [ ] 「加害者」として語っているか？
- [ ] キャッチーな造語があるか？（印鑑ラリー等）
- [ ] 対比構造があるか？（認識 vs 現実）
- [ ] 建設的な提案で終わっているか？

**エッセイ系追加チェック:**
- [ ] 笑えるポイントがあるか？
- [ ] 感動するポイントがあるか？
- [ ] 会話文が自然か？（方言のリズム）
- [ ] 比喩が意外で具体的か？
- [ ] 誰も傷つけていないか？

**専門系追加チェック:**
- [ ] 著者の専門性/経験が伝わるか？
- [ ] 専門用語に説明があるか？
- [ ] 実行可能なアドバイスがあるか？
- [ ] フレームワーク/モデルで整理されているか？

**ツールまとめ追加チェック:**
- [ ] 選定基準を明示しているか？
- [ ] 統一フォーマットでツール紹介しているか？
- [ ] 比較表があるか？
- [ ] 料金情報は最新か？

### スキ数予測セルフチェック

検索コマンド:
```bash
python .claude/skills/note-writer/scripts/search.py "スキ数 予測 [記事タイプ]" --domain publishing
```

**一般基準:**
- 80点以上: 人気記事のポテンシャルあり
- 60-79点: 質が良い、公開OK
- 40-59点: 修正してから公開を推奨
- 40点未満: 構想を練り直す

## Tips for Better Results

1. **記事タイプを最初に決める** — タイプが決まれば、タイトル・構成・技法すべてが連動する
2. **複数ドメインを検索する** — タイトル + 構成 + 技法の組み合わせで完全な設計が得られる
3. **作家スタイルを選ぶ** — 岸田奈美式/椹野道流式等の明確なスタイルで書くと一貫性が出る
4. **スキ数予測を活用する** — 公開前に必ずスコアリングでセルフチェック
5. **`--overview`で横断検索** — キーワードが複数ドメインにまたがる場合に有効
